Object-Oriented Programming

Caracteristicas de la programacion orientada a objetos

HERENCIA:

La herencia es un principio de la programación orientada a objetos que permite a una clase derivar o heredar propiedades y comportamientos de otra clase. Esto facilita la reutilización del código y la creación de jerarquías de clases. La clase que hereda se llama "clase hija" o "clase derivada", mientras que la clase de la cual hereda se denomina "clase padre" o "clase base".

POLIMORFISMO:

El polimorfismo es la capacidad de un objeto de adoptar múltiples formas. En programación orientada a objetos, esto significa que una clase hija puede sobrescribir o extender métodos de su clase base para que los mismos métodos puedan comportarse de manera diferente según el objeto que los invoque. Esto permite que el mismo código funcione con diferentes tipos de objetos de manera flexible y extensible.

ABSTRACCIÓN:

La abstracción es el proceso de simplificar la complejidad ocultando los detalles innecesarios y mostrando solo los aspectos esenciales de un objeto o concepto. En programación orientada a objetos, la abstracción se logra mediante el uso de clases abstractas e interfaces que definen un conjunto de métodos o propiedades comunes, sin especificar la implementación completa.

ENCAPSULAMIENTO:

El encapsulamiento es el principio que permite restringir el acceso a ciertos componentes de un objeto, exponiendo solo lo necesario para su uso. Esto se logra mediante modificadores de acceso como `private`, `protected` y `public`, que controlan la visibilidad y el acceso a las propiedades y métodos de un objeto. El encapsulamiento ayuda a proteger la integridad de los datos y reduce la dependencia entre diferentes partes del código.
